# Prompt — Frontend design review

Pair with `../../skills/frontend_design_review.md`. Only relevant when design/layout/CSS **is** the
task — otherwise don't touch design.

> Review the visual change against the existing design system in `css/style.css` (tokens for color,
> spacing, and type). Check:
> - Does it reuse existing tokens/components rather than adding one-off values?
> - Responsive at mobile, tablet, and desktop widths?
> - Accessible: focus rings, contrast, alt text, skip link, keyboard nav intact?
> - Consistent with the calm, warm tone of the site — nothing loud or urgent?
>
> Flag anything that drifts from the system. Suggest the minimal fix that stays on-system.

After applying: run `../checklists/regression-gate.md`, eyeball affected pages, show the diff.
